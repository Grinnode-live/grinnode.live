GRIN API v1:

The following API-Calls are GRIN API v1 Calls
  • /v1/status
    • Returns various information about the node and the network
  • /v1/blocks
    • Returns data about a specific block given a hash, a height or an unspent commit.
  • /v1/headers
    • Returns data about a block headers given either a hash or height or an output commit.
  • /v1/peers (only allowed Users)
    • Retrieves all connected peers - BAN and UNBAN not allowed
  • /v1/chain
    • Retrieves details about the state of the current fork tip
  • /v1/txhashset
    • Retrieve the roots of the TxHashSet
  • /v1/pool/push
    • Push new transaction to our local transaction pool. Add ?fluff at the end of the URL to bypass Dandelion relay.
       

Using HA-GRIN Node 

1. Setup a GRIN Wallet e.g. https://github.com/mimblewimble/docs/wiki/How-to-use-the-Grin-wallet

2. Use it as API Endpoint or add it to your grin-wallet.toml 

CLI:
           ./grin-wallet --api_server_address "https://grinnode.live:3413"

grin-wallet.toml :
          check_node_api_http_addr = "https://grinnode.live:3413"
